AcquireLeaseAsync(Int32, String, Nullable<HttpAccessConditions>, CancellationToken)

The AcquireLeaseAsync(Int32, String, Nullable<HttpAccessConditions>, CancellationToken) operation acquires a lease on the blob for write and delete operations. The lease duration must be between 15 to 60 seconds, or infinite (-1).

If the blob does not have an active lease, the Blob service creates a lease on the blob and returns it. If the blob has an active lease, you can only request a new lease using the active lease ID as proposedId, but you can specify a new duration.

For more information, see https://docs.microsoft.com/rest/api/storageservices/lease-blob.

BreakLeaseAsync(Nullable<Int32>, Nullable<HttpAccessConditions>, CancellationToken)

The BreakLeaseAsync(Nullable<Int32>, Nullable<HttpAccessConditions>, CancellationToken) operation breaks the blob's previously-acquired lease (if it exists).

Once a lease is broken, it cannot be renewed. Any authorized request can break the lease; the request is not required to specify a matching lease ID. When a lease is broken, the lease break period is allowed to elapse, during which time no lease operation except BreakLeaseAsync(Nullable<Int32>, Nullable<HttpAccessConditions>, CancellationToken) and ReleaseLeaseAsync(String, Nullable<HttpAccessConditions>, CancellationToken) can be performed on the blob. When a lease is successfully broken, the response indicates the interval in seconds until a new lease can be acquired.

A lease that has been broken can also be released. A client can immediately acquire a blob lease that has been released.

For more information, see https://docs.microsoft.com/rest/api/storageservices/lease-blob.

Note that in order to delete a blob, you must delete all of its snapshots. You can delete both at the same time using Include.

RenewLeaseAsync(String, Nullable<HttpAccessConditions>, CancellationToken)

The RenewLeaseAsync(String, Nullable<HttpAccessConditions>, CancellationToken) operation renews the blob's previously-acquired lease.

The lease can be renewed if the leaseId matches that associated with the blob. Note that the lease may be renewed even if it has expired as long as the blob has not been modified or leased again since the expiration of that lease. When you renew a lease, the lease duration clock resets.

For more information, see https://docs.microsoft.com/rest/api/storageservices/lease-blob.

SetTierAsync(AccessTier, Nullable<LeaseAccessConditions>, CancellationToken)

The SetTierAsync(AccessTier, Nullable<LeaseAccessConditions>, CancellationToken) operation sets the tier on a blob. The operation is allowed on a page blob in a premium storage account and on a block blob in a blob storage or general purpose v2 account.

A premium page blob's tier determines the allowed size, IOPS, and bandwidth of the blob. A block blob's tier determines Hot/Cool/Archive storage type. This operation does not update the blob's ETag. For detailed information about block blob level tiering https://docs.microsoft.com/en-us/azure/storage/blobs/storage-blob-storage-tiers

For more information about setting the tier, see https://docs.microsoft.com/en-us/azure/storage/blobs/storage-blob-storage-tiers.

Specifies the Uri of the source blob. The value may be a Uri of up to 2 KB in length that specifies a blob. A source blob in the same storage account can be authenticated via Shared Key.
